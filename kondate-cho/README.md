## 🚀 GitHub へのアップロード手順

1. GitHub リポジトリを開く
   → `https://github.com/soomine-svg/schedule-app/tree/main/kondate-cho`

2. **Add file → Upload files** をクリック

3. **11ファイル全てを一度にドラッグ＆ドロップ** でアップロード
   （既存ファイルは自動的に上書きされます）

4. コミットメッセージ例：
   ```
   Prep: リデザイン（配色・フォント・アイコン刷新）
   ```

5. **Commit changes** をクリック

6. GitHub Pages が数秒〜1分で自動デプロイ

## 🧹 初回アクセス時の注意（重要）

### あなた自身（既にホーム画面に追加済み）

1. **古いアイコン「献立帖」をホーム画面から削除**
2. Safari で URL を開く
3. Service Worker のキャッシュをクリア
   - Safari を完全終了 → 再起動
   - または、iPhone を再起動
4. 再度「ホーム画面に追加」
   - アイコンが **ラディッシュ（Prep）** に
   - 名前が **Prep** に

### 娘さん・知人にシェアする時

1. URL を送るだけ
2. 相手は Safari / Chrome で開く
3. 「ホーム画面に追加」
4. API キーを個別に設定（設定画面から）

## 🔍 トラブルシューティング

### Q1. アップロード後、新しいデザインが反映されない

**原因：** ブラウザ or Service Worker のキャッシュ

**対処：**
- ブラウザ：`Cmd+Shift+R`（強制リロード）
- iPhone Safari：設定 → Safari → 履歴とWebサイトデータを消去
- または iPhone を再起動

### Q2. アイコンが表示されない（タブが空になる）

**原因：** SVGファイルのアップロード漏れ

**対処：** GitHub リポジトリに5つのSVGファイル全てがあるか確認

### Q3. ホーム画面のアイコンが「献立帖」のまま

**原因：** iOS の PWA アイコンキャッシュ

**対処：**
1. ホーム画面から古いアイコンを削除
2. Safari で URL を再度開く
3. 共有 → ホーム画面に追加

### Q4. データ（レシピ・在庫）が消えた

**結論：消えません**

データは localStorage に保存されており、HTMLの更新では消えません。
万が一に備え、**念のため設定画面から「書き出し」でバックアップを取ってからアップロードすることを推奨**。

## 📝 今後の変更について

- SVG アイコンを変更したい → 該当ファイルを差し替えるだけ
- 配色を変えたい → index.html の `:root` セクションの CSS変数 を変更
- アプリ名を変えたい → manifest.json と index.html のメタタグ、ヘッダーを変更

## 💡 技術的注意事項

1. **sw.js のバージョン** が v2 → v3 になっているため、既存ユーザーも確実に最新版を取得します
2. **ネットワーク優先戦略** なので、ファイル追加時の不具合リスクは低いです
3. **manifest.json に purpose: "any"** を追加し、ホーム画面アイコンの互換性を向上
